# 新用户邮箱确认链接注册

## 背景

`email-link-auth` 已将登录入口改为邮箱密码和 OAuth，但服务端不能在密码登录失败时直接创建 `email_confirm: true` 的新用户，否则任意邮箱可在未验证归属的情况下被注册并使用。此前尝试使用邮件验证码，但当前 Supabase 免费版默认邮件服务不支持自定义模板展示验证码，因此改为 Supabase 默认邮箱确认链接。

## 目标

1. 已有账号继续使用邮箱 + 密码登录，登录过程不发送邮件。
2. 新邮箱首次注册必须通过 Supabase 默认确认链接验证邮箱归属。
3. 点击确认链接后完成注册并建立登录会话。
4. 用户注册成功后，后续登录只需要邮箱密码，不再发送确认邮件。
5. 保留 Google/GitHub OAuth 登录入口。

## 非目标

- 不新增单独注册页面。
- 不新增数据库表或外部邮件服务。
- 不改变管理员自举、用户配额或 OAuth 回调规则。

## 验收标准

1. 已存在账号输入正确密码可直接登录，且不会触发 Supabase 邮件发送。
2. 已存在账号输入错误密码返回通用错误，且不会发送确认邮件。
3. 新邮箱提交邮箱和密码后提示确认邮件已发送，点击邮件确认链接后才能完成注册。
4. 确认链接无效或过期时回到登录页并显示明确错误，不创建可用登录会话。
5. 注册完成后的同一账号可用邮箱密码再次登录，且无需再次邮件确认。
