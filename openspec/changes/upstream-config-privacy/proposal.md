# 上游配置隐私与管理员默认配置

## 背景

当前工作台的 API 配置弹窗会显示本地默认 Base URL 和 API Key，普通用户可以看到平台默认上游信息。与此同时，服务端生成接口只读取环境变量，普通用户在弹窗中填写自己的上游配置并不会真正参与生成。

## 目标

1. 普通用户默认使用平台上游，但不能查看平台默认 Base URL 或 API Key。
2. 普通用户可以在浏览器本地填写自己的 OpenAI 兼容上游配置，并用于生成、连接测试和任务重查。
3. 管理员可以在 `/admin` 页面维护平台默认上游配置。
4. 平台默认配置保存到服务端数据库，环境变量保留为兜底。

## 非目标

- 不把普通用户的自定义 API Key 保存到服务端数据库。
- 不改变现有用户配额模型。
- 不暴露平台默认 API Key 明文给前端。

## 验收标准

1. 普通用户打开 API 配置弹窗时看不到平台默认 Base URL/API Key。
2. 普通用户切换为自定义上游后，可以填写、保存并测试自己的配置。
3. 生图接口在请求包含自定义上游配置时使用该配置，否则使用平台默认配置。
4. 管理员页面可读取、测试、保存平台默认上游配置；API Key 留空时保持当前密钥。
5. Supabase 存在 `app_settings` 表，客户端角色没有读取或写入权限。
